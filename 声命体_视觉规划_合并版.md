## 1. 核心概念（已确认）

### 1.1 叙事架构：「四章节 × 四阶段」

每个章节都是**一根线的生命旅程**，遵循相同的四阶段模式：

```
开始 ──→ 发展 ──→ 高潮 ──→ 结束
(短直线)  (线变长)  (线形态)  (回归线)
  ↓         ↓         ↓         ↓
单一波形  波形展开  完整图形  收缩回线
```

**关键设计原则**：
- **不是传感器数量决定章节**，而是**每个章节内部都有这四个阶段**
- **副手是"钥匙"也是"增强器"**：解锁更高阶段上限，但不是切换章节的必要条件
- **流畅变形**：阶段之间用数学插值实现，不是离散切换

## 2. 章节详细设计

### 第一章：线生万象（Joy Division 山脉）
**当前状态**: ✅ 已实现  
**文件**: `shengmingti_proto_v3.html`

| 阶段 | 强度值 | 视觉效果 | 触发条件 |
|------|--------|----------|----------|
| **开始** | P=0.0 | 单条短直线（300px），柔和波形 | 静止或能量<30% |
| **发展** | P=0.3~0.7 | 线条变长（300→900px），层数增加（1→20层） | 主手晃动，能量>30% |
| **高潮** | P=1.0 | 30层完整山脉，最宽最长 | 副手已连接 + 双手峰值 + 能量>70% |
| **结束** | P→0 | 层数减少，长度缩短，回归单线 | 能量衰减 |

**技术实现要点**：
```javascript
// 连续变形参数（不是离散切换）
const numLayers = Math.floor(1 + P * 29);      // 1-30层连续
const lineLength = 300 + P * 600;              // 300-900px连续
const alpha = 0.1 + (i / numLayers) * 0.9;     // 每层透明度渐变

// 波形噪声（所有层统一风格）
const noise = Math.sin(p * 0.2 + t + i * 0.3) + Math.cos(p * 0.4 - t * 0.6) * 0.5;
```

**⚠️ 踩坑记录**：
- **渲染重叠问题**：最初用同一路径先stroke再fill，导致线条重叠变粗
- **解决方案**：每层只用 `ctx.stroke()`，**去掉底部填充**，避免覆盖下层线条
- **端点问题**：填充底部时 `ctx.lineTo` 到屏幕下方，会覆盖下层线条的可见性

### 3.1 BLE连接配置

| 设备 | 蓝牙名称 | UUID | 角色 |
|------|----------|------|------|
| 主手 | `ESP32` | 服务: `19b10000-e8f2-537e-4f6c-d104768a1214`<br>特征: `03504231-c554-43a4-b711-0b2945565834` | 波形生成、峰值触发 |
| 副手 | `ESP32-mini` | 同上 | 阶段增强、形态解锁 |

**⚠️ 踩坑记录**：
1. **蓝牙名称必须严格匹配**：副手固件里是 `ESP32-mini`，代码里的 `DEVICE_NAMES[1]` 必须完全一致
2. **数据结构要用数组**：之前用对象 `{main, sub}` 导致索引混乱，改为数组 `[ctrl[0], ctrl[1]]` 后稳定
3. **断开连接要清理引用**：`bleDevice` 和 `bleChar` 必须设为 `null`，否则重连会失败

### 3.2 峰值检测算法（移植自VerD）

```javascript
function detectPeak(velocity, isMain) {
  const prev = isMain ? STATE.prevMainVel : STATE.prevSubVel;
  const isRising = velocity > prev * 1.1;      // 上升沿检测
  const overThreshold = velocity > 20;          // 阈值（约2g）
  
  if (isRising && overThreshold) {
    if (isMain) STATE.mainPeaked = true;
    else STATE.subPeaked = true;
    STATE.peakHold = 25;  // 保持25帧（约400ms）
  }
}
```

**关键设计**：
- **只在上升沿触发**（甩出去的动作）
- **双峰值状态保持**：防止手抖导致峰值快速消失
- **阈值过滤桌面震动**：20（约2g）能过滤桌面轻微震动，但响应手持挥动

### 3.3 强度计算逻辑

```javascript
let target = 0;

if (hasSub && STATE.mainPeaked && STATE.subPeaked && maxE > 70) {
  // 高潮：副手 + 双峰值 + 高能量
  target = 1.0;
} else if (maxE > 30) {
  // 发展：有能量活动
  target = 0.3 + (maxE / 100) * 0.4;  // 0.3-0.7
} else {
  // 开始
  target = 0;
}

// 平滑过渡（流畅变形关键）
STATE.morphProgress += (target - STATE.morphProgress) * 0.04;
```

## 4. 渲染技术规范

### 4.1 Joy Division 山脉渲染（第一章）

```javascript
function drawChapter1(ctx, cx, cy, width, height) {
  const P = STATE.morphProgress;  // 0-1
  const numLayers = Math.floor(1 + P * 29);
  const lineLength = 300 + P * 600;
  
  // 从底部（后景）向顶部（前景）绘制
  for (let i = 0; i < numLayers; i++) {
    const y = baseY - i * spacing;  // 向上展开
    
    // 深度效果：前景亮，后景暗
    const ratio = i / (numLayers - 1);
    const alpha = 0.1 + ratio * 0.9;
    
    ctx.strokeStyle = `rgba(255, 255, 255, ${alpha})`;
    ctx.lineWidth = 1.4;  // 统一粗细！
    
    // 噪声波形（关键：所有层统一风格）
    for (each point) {
      const noise = Math.sin(p * 0.2 + t + i * 0.3) + Math.cos(p * 0.4 - t * 0.6) * 0.5;
      const amp = Math.abs(noise) * baseAmp * bell * depthAmp;
    }
    
    ctx.stroke();  // 只描边，不填充！
  }
}
```

**⚠️ 踩坑记录**：
- **不要用底部填充**：`ctx.fill()` 到屏幕底部会覆盖下层线条，导致只能看到顶层
- **线条粗细保持一致**：1.4px，不随层数变化
- **透明度渐变**：后景0.1，前景1.0，确保所有层都可见

### 4.2 流畅变形关键

1. **连续参数**：`P` 值是 0-1 的浮点数，不是离散阶段
2. **平滑插值**：`current += (target - current) * 0.04`，避免突变
3. **波形相位共享**：所有层使用相同的 `wavePhase`，确保波形连续流动
4. **从后往前绘制**：先画后景（暗、小），再画前景（亮、大），自然叠加

## 6. 技术踩坑汇总

### 6.1 渲染层

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| 底部填充覆盖 | 只能看到顶层线条，下层被黑色填充盖住 | **去掉 `ctx.fill()`**，只使用 `ctx.stroke()` |
| 线条重叠变粗 | 同一位置多条线叠加导致颜色过亮 | 每层独立路径，不共享绘制状态 |
| 端点显示但无线 | 钟形包络 `bell` 值过小导致线段断裂 | 调整包络参数 `Math.exp(-Math.pow(nx * 6, 2))` |
| 层数跳跃 | 层数从1直接跳到10，没有中间态 | 使用 `Math.floor(1 + P * 29)` 连续计算 |

### 6.2 BLE连接层

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| 副手连不上 | 蓝牙列表不显示 `ESP32-mini` | 确认固件蓝牙名称和代码 `DEVICE_NAMES` 严格一致 |
| 数据结构混乱 | 主副手数据错乱 | 使用数组 `ctrl[0]` `ctrl[1]` 替代对象 `{main, sub}` |
| 重连失败 | 断开后无法再次连接 | 断开时清理 `bleDevice = null` `bleChar = null` |

### 6.3 传感器逻辑层

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| 峰值误判 | 静止时误触发 | 增加上升沿检测 `velocity > prevVel * 1.1` |
| 峰值消失太快 | 甩动一次只触发一帧 | 增加 `peakHold` 状态保持（25帧） |
| 桌面震动触发 | 放桌上自己动 | 提高阈值到20（约2g），过滤轻微震动 |
